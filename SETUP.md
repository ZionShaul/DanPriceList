# מדריך התקנה ופריסה — מחירון משקי דן

מערכת Web/PWA בעברית (RTL) להצגת מחירי חומרי הדברה ודשן, עם מסכי מנהל לטעינת קובץ אקסל חודשי ופרסומו. בנויה ב‑Next.js 16 + Supabase, מיועדת לפריסה ב‑Vercel.

---

## 1. דרישות מקדימות
- Node.js 20+ (פותח עם 24)
- חשבון [Supabase](https://supabase.com)
- חשבון [Vercel](https://vercel.com) (לפריסה)
- חשבון [Resend](https://resend.com) לשליחת מיילי OTP (או SMTP אחר)

---

## 2. יצירת פרויקט Supabase
1. צור/י פרויקט חדש ב‑Supabase, רצוי ב‑region **EU (Frankfurt)** (קרוב לישראל).
2. ב‑**Project Settings → API** העתק/י: `Project URL`, `anon public key`, `service_role key`.

### הרצת מיגרציות (סכמה, RLS, RPC)
דרך **SQL Editor** ב‑Supabase, הרץ/י לפי הסדר את **כל** הקבצים מתיקיית `supabase/migrations/` (חשוב — בלי כולם, מסכי הניהול ייכשלו):
1. `0001_init.sql` — טבלאות, אינדקסים, RLS, פונקציות RPC, ו‑bucket אחסון.
2. `0002_seed.sql` — נתוני דמו (אופציונלי: ארגון וחומר לדוגמה).
3. `0003_publish.sql` — פונקציית פרסום/שחזור אטומית.
4. `0004_upload_title.sql` — שם תצוגה לטעינה.
5. `0005_analytics.sql` — טבלת אירועי שימוש + דוח אנליטיקס למנהל.
6. `0006_permissions_and_registration.sql` — הרשאות תצוגה למשתמש + קישור בקשת רישום.

> שים/י לב: כל מיגרציה חדשה שמתווספת לתיקייה יש להריץ גם כן. לחלופין, עם Supabase CLI: `supabase link` ואז `supabase db push` (מריץ אוטומטית את כל הקבצים).

---

## 3. הגדרת התחברות (Auth — OTP במייל)
ב‑**Authentication → Sign In / Providers**:
- ודא/י ש‑**Email** מופעל, עם **Email OTP** (קוד חד‑פעמי). אפשר לכבות "Confirm email" שאינו נחוץ ל‑OTP.
- **Authentication → Settings → SMTP**: הגדר/י SMTP מותאם של Resend (Host `smtp.resend.com`, פורט 465/587, המשתמש/סיסמה מ‑Resend). דרוש אימות דומיין השולח ב‑Resend.
- **Email Templates → Magic Link / OTP**: ודא/י שהתבנית כוללת את הקוד `{{ .Token }}` (הטקסט יכול להיות בעברית).
- **שמירת חיבור לשנה (סעיף 5 באפיון):** Supabase שומר refresh token לאורך זמן; הקובץ `proxy.ts` מרענן אותו בכל ביקור. ניתן לכוון את אורך חיי ה‑JWT תחת Settings לפי הצורך.

---

## 4. משתני סביבה
העתק/י את `.env.local.example` ל‑`.env.local` ומלא/י:

```
NEXT_PUBLIC_SUPABASE_URL=https://YOUR-PROJECT.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...        # סודי! צד שרת בלבד
SUPABASE_UPLOADS_BUCKET=uploads
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

---

## 5. יצירת מנהל ראשון
אין הרשמה עצמית — המנהל הראשון נוצר בסקריפט:

```bash
node scripts/create-admin.mjs admin@example.com "שם המנהל"
```

לאחר מכן התחבר/י דרך מסך ההתחברות עם קוד OTP שיישלח לאימייל.

---

## 6. הרצה מקומית
```bash
npm install
npm run dev
```
פתיחה: http://localhost:3000 → מועבר/ת למסך התחברות.

בדיקות לוגיקת הפרסור/החישוב:
```bash
npm test
```

---

## 7. פריסה ל‑Vercel
1. דחוף/י את הקוד ל‑GitHub.
2. ב‑Vercel: **New Project** → ייבוא הריפו.
3. הוסף/י את משתני הסביבה (סעיף 4) ב‑**Settings → Environment Variables**. עדכן/י `NEXT_PUBLIC_SITE_URL` לכתובת הפרודקשן.
4. Deploy. Vercel מספק HTTPS אוטומטי, זמין מכל רשת בארץ.
5. **דומיין מותאם (אופציונלי):** הוסף/י דומיין (למשל `pricelist.example.co.il`) תחת Domains.
6. ב‑Supabase **Authentication → URL Configuration** הוסף/י את כתובת הפרודקשן ל‑Redirect URLs / Site URL.

## 8. פורמט קובץ האקסל
השורה הראשונה היא כותרות. המערכת מזהה את העמודות הבאות (כולל שמות נרדפים נפוצים):

| שדה | כותרות מזוהות |
|-----|----------------|
| לקוח | לקוח / שם לקוח / ארגון |
| ספק | ספק / שם ספק |
| מק״ט | מק״ט / מקט / קוד פריט |
| תיאור מוצר | תיאור מוצר / תיאור / שם חומר / פריט |
| כמות | כמות |
| מחיר ליחידה | מחיר ליחידה / מחיר יחידה / מחיר |
| סה״כ מחיר | סה״כ מחיר / סכום / סך הכל |
| תאריך | תאריך |
| מספר חשבונית/תעודה | חשבונית / תעודה / אסמכתא |

**חוקי חריגה (לא נכללים בחישוב, מוצגים למנהל):** מחיר≤0, כמות≤0, שורת זיכוי/החזר, חוסר תיאור/לקוח, או שורה שלא ניתן לחשב ממנה מחיר.

---

## 9. זרימת עבודה של מנהל
1. **ניהול → טעינת קובץ**: בחירת אקסל → המערכת מנתחת ומציגה סיכום (תקינות/חריגות/חומרים לא ממופים). הנתונים עדיין אינם גלויים למשתמשים.
2. **קטלוג חומרים**: מיפוי חומרים לא ממופים לחומר תקני (אופציונלי).
3. **פרסום**: לחיצה על "פרסום" הופכת את הטעינה לפעילה היחידה.
4. **היסטוריה**: שחזור טעינה קודמת או מחיקת טעינות שאינן הפעילה.

---

## הערה על אבטחה
- בידוד נתוני ארגון נאכף ב‑Postgres RLS — משתמש רגיל רואה רק את רכישות הארגון שלו.
- מחירים כלליים מחושבים דרך פונקציות `SECURITY DEFINER` המחזירות **אגרגציה בלבד**, ללא חשיפת שורות של ארגונים אחרים.
- `SUPABASE_SERVICE_ROLE_KEY` הוא סוד צד‑שרת בלבד — לעולם לא נחשף לדפדפן.
